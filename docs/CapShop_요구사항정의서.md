## 1. 문서 개요

### 1.1 목적

본 문서는 CapShop 서비스의 기능 요구사항과 비기능 요구사항을 정의하기 위한 문서이다.  
CapShop은 사용자가 YouTube 영상 시청 중 발견한 상품을 직접 검색하지 않아도, Chrome Extension으로 영상 속 상품 영역을 캡처하면 AI가 이미지와 텍스트 정보를 분석하고 유사 상품을 추천해주는 쇼핑 추천 서비스이다.

### 1.2 범위

본 요구사항정의서는 다음 범위를 포함한다.

- 사용자 회원가입, 로그인, 소셜 로그인, 계정 찾기
- 상품 목록, 상품 상세, 베스트 상품, 카테고리 상품 조회
- YouTube 영상 기반 상품 캡처 Chrome Extension
- 캡처 이미지 기반 AI 상품 분석
- 네이버 쇼핑 검색 기반 상품 후보 수집
- AI 기반 검색어 생성, 후보 필터링, 이미지 유사도 리랭킹
- 추천 상품 표시 및 구매 링크 이동
- 위시리스트, 최근 조회 내역, 사용자 행동 로그
- 관리자 대시보드, 사용자 관리, 게시글 관리
- 데이터베이스 및 외부 API 연동
- 보안, 성능, 운영, 확장성 등 비기능 요구사항

## 2. 프로젝트 개요

### 2.1 서비스 한 줄 정의

영상 속 상품을 캡처하면 AI가 상품을 분석하고, 유사한 쇼핑 상품을 추천해주는 AI Commerce Recommendation 서비스이다.

### 2.2 추진 배경

YouTube, Shorts 등 영상 콘텐츠를 시청하다 보면 사용자가 관심 있는 옷, 가방, 전자기기, 소품 등을 발견할 수 있다.  
그러나 영상 속 상품은 브랜드명, 모델명, 정확한 상품명을 알기 어렵기 때문에 사용자는 직접 화면을 멈추고 색상, 형태, 로고, 분위기 등을 추측해 검색해야 한다.

CapShop은 이 과정을 줄이기 위해 다음과 같은 흐름을 제공한다.

1. 사용자가 YouTube 영상에서 원하는 상품 영역을 드래그하여 캡처한다.
2. AI가 캡처 이미지를 분석하여 상품 특징을 추출한다.
3. OCR 결과와 시각 특징을 바탕으로 검색어를 생성한다.
4. 네이버 쇼핑 검색과 Gemini 기반 분석을 통해 후보 상품을 수집한다.
5. 후보 상품을 필터링하고 리랭킹하여 사용자에게 추천한다.

### 2.3 서비스 목표

| 목표 | 설명 |
|---|---|
| 검색 진입 장벽 감소 | 상품명을 몰라도 이미지 캡처만으로 검색 가능하도록 한다. |
| 영상 시청 흐름 유지 | 사용자가 콘텐츠 시청 중 자연스럽게 상품 탐색을 이어갈 수 있도록 한다. |
| 추천 정확도 향상 | OCR, 이미지 특징, 검색 결과를 함께 활용하여 유사 상품을 추천한다. |
| 사용자 행동 기반 확장 | 조회, 클릭, 위시리스트, 구매 이동 로그를 저장하여 향후 개인화 추천으로 확장한다. |
| 직관적인 UX 제공 | Chrome Extension과 마스코트 UI를 통해 분석 상태를 쉽게 이해할 수 있도록 한다. |

## 3. 시스템 구성

### 3.1 전체 시스템 구성

| 구성 요소 | 기술 | 주요 역할 |
|---|---|---|
| Frontend Web | Vue 3, Vite, Vue Router, Pinia, Axios | 사용자 화면, 관리자 화면, 상품 탐색, 마이페이지 |

| Cache / Token Store | Redis | 토큰 블랙리스트, 캐시 |

## 5. 전체 서비스 흐름

### 5.1 YouTube 캡처 기반 추천 흐름

```text
1. 사용자가 YouTube 영상을 시청한다.
2. Chrome Extension이 YouTube 영상 페이지를 감지한다.
3. 사용자가 CapShop 캡처 버튼을 클릭한다.
4. 로그인 상태가 아니면 로그인 패널을 표시한다.
5. 사용자가 영상 속 상품 영역을 드래그하여 선택한다.
6. Extension이 현재 탭 이미지를 캡처하고 선택 영역만 crop한다.
7. 사용자가 빠른 검색 또는 정밀 검색을 선택한다.
8. Extension이 Backend의 분석 API로 videoId, timestampSec, imageBase64, searchMode, userHint를 전송한다.
9. Backend가 AI Server에 분석을 요청한다.
10. AI Server가 OCR, 이미지 특징, 상품 후보명, 검색어를 생성한다.
11. 네이버 쇼핑 검색 및 Gemini 기반 분석을 통해 상품 후보를 수집한다.
12. 후보 상품을 필터링하고 필요 시 이미지 유사도 기준으로 재정렬한다.
13. Backend가 추천 결과를 Extension에 반환한다.
14. Extension이 결과 패널에 AI 탐지 결과와 추천 상품을 표시한다.
15. 사용자가 상품을 클릭하거나 판매 페이지로 이동하면 행동 로그를 저장한다.
```

### 5.2 일반 웹 상품 탐색 흐름

```text
1. 사용자가 웹 메인 화면에 접속한다.
2. 상품 목록, 베스트 상품, 카테고리 상품을 조회한다.
3. 상품 상세 화면에서 상세 정보와 관련 상품을 확인한다.
4. 로그인 사용자는 상품을 위시리스트에 추가하거나 최근 조회 내역을 확인한다.
5. 사용자 행동 로그는 향후 개인화 추천을 위해 저장된다.
```

### 8.2 데이터 저장 원칙

| 구분 | 요구사항 |
|---|---|
| 사용자 데이터 | 이메일은 unique해야 하며, 소셜 로그인은 provider와 provider_id 조합으로 구분한다. |
| 상품 데이터 | 외부 검색 상품은 source와 external_product_id로 출처를 구분한다. |
| 이미지 데이터 | 캡처 원본은 직접 저장하지 않고 image_hash 또는 분석 결과 중심으로 저장할 수 있다. |
| 행동 로그 | 이벤트 유형, 발생 위치, 상품, 추천, 영상 정보, 메타데이터를 함께 저장한다. |
| AI 로그 | 요청/응답 JSON, 성공 여부, 에러, 응답 시간을 저장하여 분석 품질 개선에 활용한다. |
| 삭제 정책 | 사용자 삭제 시 사용자 관련 선호, 탐지, 추천, 위시리스트, 이벤트는 cascade 또는 set null 정책에 따라 정리한다. |

## 12. 예외 및 오류 처리 요구사항

| 상황 | 처리 방식 |
|---|---|
| YouTube videoId를 찾을 수 없음 | 사용자에게 영상 ID를 찾을 수 없다는 오류 메시지를 표시한다. |
| 영상 timestamp를 찾을 수 없음 | 사용자에게 영상 정보를 찾을 수 없다는 오류 메시지를 표시한다. |
| 선택 영역이 너무 작음 | 분석 요청을 중단하고 다시 선택하도록 안내한다. |
| 로그인하지 않음 | 로그인 패널을 표시하고 인증 후 분석을 계속한다. |
| Access Token 만료 | 저장된 토큰을 제거하고 재로그인을 요구한다. |
| Backend 연결 실패 | 서버 연결 오류 메시지를 표시한다. |
| AI 분석 실패 | 분석 실패 메시지와 재시도 안내를 표시한다. |
| 외부 검색 결과 없음 | 표시할 상품이 없다는 메시지를 표시한다. |
| 상품 이미지 URL 오류 | 해당 이미지를 제외하거나 대체 표시하여 전체 결과 렌더링을 유지한다. |
| 외부 판매 링크 없음 | 링크 비활성화 또는 이동 방지 처리를 한다. |

## 15. 제약사항 및 가정

### 15.1 제약사항

- Chrome Extension은 YouTube 페이지를 주요 대상으로 한다.
- AI 분석 품질은 Gemini API와 이미지 품질, OCR 가능 여부에 영향을 받는다.
- 네이버 쇼핑 검색 결과는 외부 API 응답 품질과 가용성에 의존한다.
- 외부 상품 이미지 URL은 만료되거나 접근이 차단될 수 있다.
- 운영 환경에서는 localhost 및 내부 IP 기반 설정을 환경별로 분리해야 한다.

### 15.2 가정

- 사용자는 상품명을 모르는 상황에서도 이미지 캡처를 통해 검색하고자 한다.
- 추천 결과는 정확한 동일 상품 보장보다는 유사 상품 추천을 목표로 한다.
- 사용자 행동 로그는 향후 개인화 추천을 위해 지속적으로 축적된다.
- 관리자 기능은 사용자/게시글 관리 중심으로 우선 제공한다.
- AI 분석 결과는 탐지 정확도와 검색어 품질 향상을 위해 로그로 관리한다.

## 16. 인수 기준

| 구분 | 인수 기준 |
|---|---|
| 캡처 분석 | YouTube 영상에서 상품 영역을 캡처하고 AI 분석 요청이 성공해야 한다. |
| 추천 결과 | 분석 결과를 바탕으로 최소 1개 이상의 유사 상품 후보를 표시할 수 있어야 한다. |
| 검색 모드 | 빠른 검색과 정밀 검색 요청값이 구분되어 처리되어야 한다. |
| 인증 | 로그인하지 않은 사용자는 분석 요청 전 로그인 안내를 받아야 한다. |
| 상품 이동 | 추천 상품의 판매 페이지 링크 클릭 시 새 탭으로 이동해야 한다. |
| 로그 저장 | 상품 클릭과 구매 링크 클릭 이벤트가 사용자 행동 로그로 저장되어야 한다. |
| 관리자 | 관리자 계정은 사용자 관리와 게시글 관리 화면에 접근할 수 있어야 한다. |
| 오류 처리 | 분석 실패, 서버 오류, 인증 만료 상황에서 사용자가 이해할 수 있는 메시지를 받아야 한다. |
